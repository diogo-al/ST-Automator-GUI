# ST-Automator GUI

An automated testing engine developed in Java to streamline the validation of algorithm implementations (specifically targeted at AED/Algorithms and Data Structures tasks).

## 🚀 Key Features
* **Automated Execution:** Runs test cases against a specific Java class ("ST") to ensure logic consistency.
* **Interactive GUI:** Built with Java Swing for a more intuitive user experience compared to CLI-only tools.
* **Gamified Progress:** Includes a built-in reward/point system to track debugging milestones and successful test runs.
* **Real-time Validation:** Provides immediate feedback on whether the code meets the required output patterns.

### 🛡️ Security & Logic Implementation
* **Binary Persistence:** To prevent users from easily modifying their "bank" balance, I implemented binary file I/O. This ensures that the data is obfuscated and requires a specific parser to read/write, adding a layer of integrity to the local state.
* **Secret Command Engine:** Integrated a hidden cheat-code system that validates specific string patterns to unlock developer-mode features.
* **Session Management:** Uses a "Casino" metaphor to manage virtual currency, teaching core concepts of transaction logic and state persistence.
